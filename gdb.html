Debugging

CS202 Lab




GDB (GNU Debugger)



Basic Commands:


  • gdb <name_of executable>- Loads your exexutable with GDB.
  • r <list of args> - Begins running your program.
  • set args <args list> - set args list for next run.
  • show args - output the current list of arguments.
  • q or quit - Terminates your program/debugging session.
  • kill - kill the current program.
  • bt - backtraces the prgram stack (VERY HELPFUL WITH SEG FAULTS!!!)
  • p <variable name> - prints the value stored in that variable.
  • p <*name of pointer> - prints the value being pointed to.


Breakpoints:


  • b or break can be used to set breakpoints
  • b file:line - sets a breakpoint at the line in the file specified.
  • b file:function - sets a breakpoint at the function in the file specified.
  • b <address> - sets a breakpoint at the adress specified from the executable.
  • s - steps through your program executing one line at a time.
  • n - (next line), steps over function calls.
  • c - continue executing program normally after break has been encountered.
  • info break - ouputs all set breakpoints with coorresponding breakpoint numbers.
  • delete - deletes all breakpoints.
  • delete n - deletes breakpoint number n.
  • disable - disables all breakpoints.
  • disable n - disables breakpoint number n.
  • enable - enables all breakpoints.
  • enable n - enables breakpoint number n.


Note:
For more GDB commands please refer to the man pages by typing in "man gdb" in your Terminal or Putty session or click here for the supplimetal lab handout.
